Guidelines

With rubico, your code can take many directions; the following recommendations are not meant to be followed strictly, but can help you better reason about your programs if followed when possible.

  1. Write pure functions - your functions should avoid side effects and shared state.
  2. Write unary functions - your functions should take one argument.
  3. Prefer partial application over variables - write higher ordered functions instead of creating higher-scoped variables.
